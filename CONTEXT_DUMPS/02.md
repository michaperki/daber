# Context Dump 02 — Daber App Consolidation, New Screens, Audio Orchestration, and Next Steps

Date: 2026-03-15

## Overview
We migrated the project to a single Next.js app under `Daber/`, removed duplicated root app/lib/prisma/data code, implemented key V2 screens (Dashboard, Library, Progress, Retry Missed, Vocab Flashcards), and tightened the drill orchestration (no more double‑prompt playback; optional prompt prefix stripping after the first item).

## What Changed (Highlights)
- Single app: Daber is now the canonical frontend + API.
  - Root package.json scripts delegate to Daber (dev/build/start, prisma scripts).
  - Removed root `app/*`, `lib/*`, `prisma/*`, and `data/*` duplicates to avoid confusion.
- Database and content under Daber:
  - `Daber/prisma/*` (schema, seed) and `Daber/data/lessons/*` (present_tense_basics_01.json).
- Drill improvements:
  - Added per‑item, one‑time prompt playback guard to prevent double TTS in dev Strict Mode.
  - Strips leading “How do I say …” after the first item in a session (for both display and TTS) to keep prompts lean.
  - Centralized audio logic via hooks: `useTTS`, `useMicRecorder`.
- Screens implemented:
  - Dashboard `/` (home): last/in‑progress session progress, weekly grid, stats, missed snippets, quick start.
  - Library `/library`: lists lessons from Prisma with Start buttons.
  - Progress `/progress`: KPIs + stacked bars for 7d/30d/all‑time, mastery by lesson, hardest items.
  - Retry Missed `/retry`: groups recent misses by lesson with quick Start.
  - Vocab `/vocab`: flashcards parsed from `Mike_Hebrew_Vocab.md` with reveal/hear/next.
- Styles consolidated in `Daber/app/globals.css` to match the model_frontend mocks.

## Files of Interest
- Daber/app/page.tsx — Dashboard (server) + StartOrContinueButton (client).
- Daber/app/library/page.tsx — Lesson list (server) using Prisma.
- Daber/app/progress/page.tsx — Progress (server) computes datasets.

- Daber/app/api/* — Endpoints re‑scoped under Daber.

## Remaining Screens / Gaps
- Profile page (from model_frontend/hebrew_profile_screen.html)
  - Not yet implemented. Should show avatar/identity stub, all‑time stats, quick links, and possibly UI settings.
- Quick Nav (bottom nav bar) is present in mocks; we use simple links per page.
  - Optionally add a light footer nav component mirrored across pages.
- Retry‑Missed fidelity
  - Currently starts the full lesson containing recent misses. A more surgical “review” mode (only missed subset) would need a new workflow.

## Behavior Notes
- “How do I say…” stripping
  - The first item in a session uses the full prompt; subsequent items strip a leading “How do I say” (with punctuation) before display and TTS replay.
- Strict Mode dev
  - One‑time guard for prompt TTS (keyed by lesson item id) prevents double playback if effects fire twice.
- STT/TTS
  - No mocks; requires `OPENAI_API_KEY`. Endpoints return 500 with clear messages if keys are missing.

## Next Steps (Implementation Plan)
1) Profile Page
- Route: `Daber/app/profile/page.tsx` (server) with a small client card for settings (if needed).
- Content: basic identity stub (avatar initial), total sessions/items/accuracy, links to Library/Progress/Vocab.
- Optional: UI settings shell (e.g., “strip ‘how do I say’ always”, “stay on flawed”, “show transliteration”). Persist to localStorage.

2) Retry Missed, Focused Mode
- Option A: “Review Session” endpoints
  - POST `/api/review/sessions` with a list of `lesson_item_ids` → returns review session id.
  - GET `/api/review/sessions/:sessionId/next-item` returns next item from the set.
- Option B: Query parameter
  - Reuse session endpoints but allow `create session` with a `subset: lesson_item_ids[]` stored on Session (requires DB change or transient storage).
- UI: On `/retry`, add “Drill selected misses only” which collects 6–12 most recent misses and starts a review session.

3) Progress Page Enhancements
- Enable clicking bars to filter the list of missed/attempts below.
- Add small “Top lessons by accuracy” component.
- Add client toggles for 7d/30d/all (already done), persist last selected range (localStorage).

4) Dashboard
- “retry missed” button currently links to `/retry`; optionally preselect the top missed pack and start review mode (from step 2).
- Consider showing a tiny feed of last 3 attempts with grade colors.

5) Drill UX
- Settings persistence (localStorage): stay‑on‑flawed, show transliteration, always drop “how do I say”.
- Device picker caching by label/deviceId where supported.
- Debounce replay prompt clicks (current usage relies on arming guard; optional).

6) Content / Authoring
- Import from `Mike_Hebrew_Vocab.md` to a “Vocab Items” table and provide a simple authoring UI.
- Author new packs; run evaluator tests (`scripts/test_evaluator.ts`) before seeding.

7) Technical Cleanup
- Add a small footer nav component shared on Dashboard/Library/Progress/Profile for parity with mocks.
- Add basic Zod request validation for API POST bodies.
- Add 1–2 integration tests for evaluator or a CLI to run quick grading checks on content.

## Risks / Dependencies
- Review mode likely needs a small DB or API change for subset sessions.
- STT/TTS require stable network and keys; local dev without them fails as designed.

## Quick Commands
- Dev: `npm run dev` (delegates to Daber)
- DB push/seed (in env): `npm run db:push` and `npm run seed`
- Evaluator sanity: `npm run test:evaluator`

## Summary
We consolidated the app under Daber, removed duplication, and implemented the redesigned UI for core screens with typed server data. The drill flow is smoother (single prompt playback, less verbose prompts after the first item). Next, we’ll build the Profile page, add a true “retry missed (subset)” mode, and refine Progress/Dashboard interactions, while persisting user preferences.
