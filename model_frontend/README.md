# Model Frontend (Canonical Visuals)

Purpose
- This folder contains static HTML mockups generated by an LLM.
- Treat these files as the canonical visual reference for building the new Daber app.
- Do not ship or import these files directly into the app; they are design references only.

How to use
- Mirror the layout, spacing, and states closely when building React components.
- Reuse the typography (DM Sans, DM Serif Display, DM Mono) — include the same Google Fonts in the app layout.
- Match key UI sections and states:
  - Drill screen: prompt card, hint toggle, status strip, mic controls, transcript preview, feedback card, and next button.
  - Home/dashboard, library, progress, profile, and session summary later.
- Use the color and style semantics via CSS variables where possible (e.g., `--color-text-primary`, `--color-background-primary`, border radii tokens, etc.).

Notes
- The files are static and standalone; the real app is React/TypeScript and uses runtime data and APIs.
- V1 endpoints (e.g., `/api/sessions`, `/api/attempts`, `/api/stt`, `/api/tts`) are the source of truth for initial functionality until Daber back-end evolves.
- When there is a discrepancy, prefer these visuals, but keep API contracts from `lib/contracts.ts` stable.

Files
- `hebrew_drill_screen.html` — primary reference for the drill session UI states.
- `hebrew_home_dashboard.html` — reference for the home screen sections and cards.
- `hebrew_lesson_library.html` — reference for the packs list with filters.
- `hebrew_progress_screen.html` — reference for charts and mastery lists.
- `hebrew_profile_screen.html` — reference for user settings/toggles and trophies.
- `hebrew_session_summary.html` — reference for session result visualization and follow-up actions.
