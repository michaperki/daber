# Hebrew Drill Teacher — Context Dump 01

This document captures the initial V1 prototype I implemented: tech stack, data, endpoints, UI/loop behavior, evaluator, logging, audio pipeline, and next steps.

## Summary
- Core loop: Prompt (English) → Speak/Type answer → STT → Evaluate (Correct/Flawed/Incorrect) → Teacher says correct Hebrew → Auto‑advance to next item.
- Built as a thin Next.js + Prisma + Postgres app with OpenAI Whisper (STT) and gpt‑4o‑mini‑tts (TTS).
- Single lesson pack of 10 “Present Tense Basics” items seeded into the DB.

## Stack
- Frontend: Next.js 14 (App Router), TypeScript, minimal CSS.
- Backend: Next.js API routes, Prisma ORM, Postgres.
- STT: OpenAI Whisper (`whisper-1`) via `/api/stt`.
- TTS: OpenAI `gpt-4o-mini-tts` via `/api/tts` with simple in‑memory caching + prefetch.

## Setup
1. Env
   - Copy `.env.example` to `.env` and set:
     - `DATABASE_URL` for Postgres
     - `OPENAI_API_KEY` for STT/TTS
2. Install: `npm install`
3. Prisma: `npm run prisma:generate` and `npm run db:push`
4. Seed lesson: `npm run seed`
5. Dev: `npm run dev` then open http://localhost:3000/drill

## API Endpoints
- `POST /api/sessions` → create session
  - Body: `{ lessonId }`
  - Returns: `{ session: { id, lesson_id, started_at } }`
- `GET /api/sessions/:sessionId/next-item`
  - Returns: `{ done, item?, index?, total? }`
  - `index` = number attempted + 1; `total` = total lesson items
- `POST /api/attempts` → record attempt + grade
  - Body: `{ sessionId, lessonItemId, rawTranscript }`
  - Returns: `{ grade, reason[], correct_hebrew }`
- `GET /api/sessions/:sessionId/summary`
  - Returns: session counters + total attempts
- `POST /api/stt` → transcribe audio (WebM/Opus)
  - Multipart `audio` (Blob) → Whisper
  - Also accepts `{ text }` passthrough (unused in current flow)
- `POST /api/tts` → synthesize speech for given text
  - Returns audio/mpeg; cached in‑memory by exact text

## Evaluator
- Files: `lib/evaluator/normalize.ts`, `lib/evaluator/deterministic.ts`, `lib/evaluator/index.ts`
- Steps
  - Normalize transcript (strip punctuation/diacritics, collapse spaces, lowercase where relevant)
  - Deterministic: exact target or accepted variants → Correct
  - Near‑miss authored examples → Flawed (e.g., wrong gender, pronoun omission)
  - Tiny edit distance (<=1) → Flawed
  - Else → Incorrect
- Tests: `npm run test:evaluator` (script in `scripts/test_evaluator.ts`)

## Logging
- Structured JSON logs to console + persisted in `Event` table via `lib/log.ts`.
- Events: `session_started`, `attempt_graded`, `session_ended`, `stt_transcribed`, `stt_text_passthrough`, `tts_generated`.

## UI / Loop Behavior
- Pages

  - Drill setup: `app/drill/page.tsx`
  - Session: `app/session/[sessionId]/page.tsx`
  - Summary: `app/session/[sessionId]/summary/page.tsx`
- State machine (client)
  - Phases: `listen` → `answer` → `feedback`
  - Top step header shows current phase; progress bar + counter `index / total`
  - Buttons disabled by phase (e.g., no submit while TTS playing or evaluating)
- Auto‑advance
  - On submit: play correction TTS to completion, dwell ~800ms, then advance (regardless of grade)
  - Previous result banner persists on the next prompt with “Replay correction”
- Speak vs Type
  - Segmented control: Speak (default) | Type
  - Speak: mic controls, transcript preview; Type: input only
- Keyboard shortcuts
  - Enter = submit (when not typing)
  - Space = replay prompt (when not typing, not recording)

## Audio Pipeline
- Prompt/Correction TTS
  - Server `/api/tts` using `gpt-4o-mini-tts`
  - Single‑audio channel; cancels existing audio before new playback
  - TTS caching (Map<text, Blob>) + prefetch next prompt while correction plays
  - 500ms arming delay after TTS to avoid mic capturing prompt tail
- Mic capture (Speak mode)
  - MediaRecorder WebM/Opus + device picker + echo cancellation/noise suppression/AGC
  - Live audio level meter via Web Audio Analyser (RMS)
  - VAD: stop on ~900ms silence after speech; 7s hard cap
  - POST audio to `/api/stt` (Whisper); transcript fills input preview for review
