## Prompt Behavior (Current)
- Always shows the English prompt (defaults in settings updated accordingly).
- Removes leading “How do I say” (with punctuation) from display and prompt TTS.
- Prompt TTS is off by default; enable in Profile if desired (replay button appears when enabled).

## SVG/Icon Sizing Fix
- Quick Start icons now set explicit `width/height` and CSS sizing for consistent rendering.

## How to Verify E2E
1. `npm run db:push` (adds `subset_item_ids`).
2. `npm run seed` (imports “My Vocab 01” from Mike_Hebrew_Vocab.md; shows count).
3. `npm run dev`.
4. Home main tile → starts a `user_vocab_01` session; English prompt shows; submit and see correction.
5. Retry → select items per lesson and “drill selected”; session ends after selected items.
6. Profile → adjust settings (optional); verify transliteration + stay-on-flawed behavior.

- Settings simplification: remove “show English prompt text” and “hide single-word prompts” toggles (prompt is always shown per user preference), keep only “speak English prompt (TTS)” + “show transliteration” + “stay on flawed”.
- API validation: add Zod request schemas for `/api/attempts` and `/api/stt` JSON body.
- Mic device persistence and selection UI (cache device by label/deviceId; dropdown in session).
- Retry filters: quick scopes for last 7/30 days; “top N misses” preset buttons.
- Progress/Dashboard: click bars to filter lists; small attempt feed on Home.
- Evaluator: expand near-miss patterns; optional transliteration acceptance set; integration test for subset flow.
- Content extras: optional transliteration parse from MD; authoring UI for lesson items (vocab, grammar packs).
- Cleanup: remove root `app/v2/*` and `lib/client/state/sessionMachine.ts` (unused at root) after confirming no references; update README/V2 plans to reflect single-app roadmap.
- DB indexes: consider indexes on `Attempt(lesson_item_id)` and `Attempt(session_id)` for groupBy/count performance.
